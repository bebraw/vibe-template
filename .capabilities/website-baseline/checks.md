# Checks

After applying the website baseline kit:

- Confirm the copied checklist names its owner or owning spec.
- Confirm every conditional section is marked applicable or not applicable.
- Verify deterministic requirements through the target repo's existing tests where practical.
- Run Lighthouse or the project's equivalent browser audit when available.
- Run the target repo's normal quality gate.
- Manually check keyboard navigation, visible focus, zoom, reduced motion, error recovery, and meaningful content without client JavaScript.

Do not mark legal, privacy, accessibility, or security compliance solely from automated results.
